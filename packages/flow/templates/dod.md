# Definition of Done

> **Status**: Stub — pull from the company's current standard and replace this file.
> Until confirmed, the `/plan` and `/create-pr` skills will reference this as a reminder checklist only.

---

A User Story is **Done** when:

- [ ] All acceptance criteria are met and verified
- [ ] Code reviewed and approved by at least one peer
- [ ] Tests written and passing (unit + integration where applicable)
- [ ] No new linting or type-check errors introduced
- [ ] PR linked to the ADO work item (`AB#id`)
- [ ] Work item state set to "Done" in ADO
- [ ] Deployed to the relevant environment (if applicable)
- [ ] No outstanding TODO comments left in the changed code
